# Regression Risk Register

| Area                        | Current risk                                                                               | Why it matters                                               | Recommended protection                                                                             |
| --------------------------- | ------------------------------------------------------------------------------------------ | ------------------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| Bootstrap introduction      | Global resets and class-name overlap alter Bulma layout/forms                              | Existing UI depends heavily on Bulma classes                 | Computed-style diff, Playwright desktop/mobile snapshots, no Bootstrap reboot at first             |
| CSS loading                 | Deferred CSS import is documented as optimized away in production                          | Future classes may silently miss production CSS              | Build smoke test plus visual test for classes only present in deferred CSS before changing styling |
| URL hash compatibility      | Hash parsing resolves aliases, variants, recolors, and legacy names                        | Shared URLs are a core persistence format                    | Unit fixtures for old/new hashes and round-trip sync                                               |
| Default initialization      | Metadata readiness, hash hydration, canvas init, and defaults are sequenced asynchronously | Race conditions can produce empty previews or wrong defaults | Browser test for cold load, hash load, and default render completion                               |
| Global mutable state        | Character, UI, preview, and export state share one object                                  | UI changes can accidentally trigger renders/exports          | Action-level tests for selection/reset/body type without UI mounting                               |
| Selection grouping          | One selection per `type_name`; recolor subgroups can target other type names               | Incorrect grouping breaks layered character composition      | Unit tests for item and sub-selection grouping                                                     |
| Body color matching         | Changes propagate to `matchBodyColor` selections                                           | Important creator convenience and visual consistency         | Unit tests for enabled/disabled body color matching and sub-recolor behavior                       |
| Layer order                 | z-position sort happens in renderer and metadata helpers                                   | Preview/export can differ if order logic drifts              | Golden render-plan tests and ZIP fixture checks                                                    |
| Animation support mapping   | Some UI animation names map to metadata aliases                                            | Items may disappear from certain rows/exports                | Unit tests for combat/backslash/halfslash matching                                                 |
| Sprite path resolution      | Depends on variants, recolors, body type, templates, and layers                            | Missing paths cause invisible sprites                        | Unit tests around `getSpritePath` and template replacement                                         |
| Palette recoloring          | WebGL/CPU/fallback paths can differ                                                        | Recolored output is central to character identity            | Pixel/golden tests for representative recolors                                                     |
| Custom animations           | Wheelchair/custom rows append below the standard sheet                                     | Preview and ZIP exports rely on custom y positions           | Tests for custom animation draw areas and export slices                                            |
| Preview animation lifecycle | Component starts/stops RAF and pinch handlers                                              | Leaks or duplicate loops hurt performance                    | Browser lifecycle test for mount/update/unmount                                                    |
| Zoom/pinch                  | Touch gestures interact with nested scroll containers                                      | Mobile usability depends on it                               | Mobile Playwright interaction smoke test                                                           |
| PNG export                  | Downloads current offscreen canvas                                                         | Primary output workflow                                      | Canvas/download test with known dimensions/blob type                                               |
| ZIP export modes            | Heavy workflows use globals and async canvas operations                                    | Users rely on split animation/item/frame outputs             | Existing issue 382 fixtures plus tests for all modes after UI changes                              |
| Credits export              | Credits TXT/CSV gathered from selected catalog items                                       | License compliance depends on it                             | Unit tests for credits aggregation and generated files                                             |
| Clipboard JSON              | Import/export mutates full state object                                                    | Saved-character transfer can break silently                  | Round-trip JSON tests with version/missing-field cases                                             |
| Search/filtering            | Query, license, and animation filters shape tree visibility                                | Creator navigation depends on it                             | Derived-result tests independent of markup                                                         |
| Accessibility               | Collapsibles/tree/canvas/loading statuses need keyboard/ARIA review                        | Bootstrap migration can improve or regress a11y              | Axe/manual keyboard checklist for panels, tree, export buttons, canvas labels                      |
| Mobile layout               | Nested scroll, sticky header, dense controls                                               | Main UX risk for modernization                               | Mobile visual baseline and touch-target audit                                                      |
| Performance                 | Full renders load/recolor many images; tree/variant UI can be large                        | Bootstrap animations/layout changes may add jank             | Render timing profile, zip profile, avoid large collapse animations                                |
